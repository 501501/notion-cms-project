## 1. 프로젝트 개요

### 1.1 목적

Notion을 CMS(Content Management System)로 활용하여 개인 프로젝트 포트폴리오 웹사이트를 구축합니다.

### 1.2 가치 제안

- **자동화된 콘텐츠 관리**: Notion에서 프로젝트 정보를 입력/수정하면 웹사이트에 자동으로 반영
- **유지보수 용이성**: 코드 수정 없이 Notion에서 포트폴리오 관리
- **확장성**: Notion API를 활용한 유연한 데이터 구조
- **전문성**: 현대적이고 반응형 포트폴리오 웹사이트로 개발자 이미지 강화

### 1.3 타겟 사용자

- 웹/모바일 개발자
- 디자이너
- 포트폴리오를 효율적으로 관리하고자 하는 개인

## 4. Notion 데이터베이스 구조

### 4.1 프로젝트 데이터베이스 스키마

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| `title` | Title | ✅ | 프로젝트 제목 |
| `category` | Select | ✅ | 프로젝트 카테고리 (웹/모바일/디자인) |
| `description` | Rich Text | ✅ | 프로젝트 설명/개요 |
| `tags` | Multi-select | - | 관련 기술/도구 태그 |
| `publishedDate` | Date | ✅ | 프로젝트 완료 또는 공개일 |
| `image` | File | - | 프로젝트 썸네일 이미지 |
| `content` | Rich Text | - | 상세 프로젝트 내용 |
| `externalLink` | URL | - | 외부 링크 (GitHub, Demo 등) |
| `featured` | Checkbox | - | 홈페이지에 노출할 주요 프로젝트 여부 |
| `status` | Select | ✅ | 상태 (진행중/완료/보관) |

### 4.2 데이터베이스 필터 및 정렬

- **기본 정렬**: `publishedDate` 내림차순 (최신순)
- **필터링 옵션**:
  - 상태 = "완료"
  - featured = true (홈페이지용)

## 5. 페이지 및 라우팅

### 5.1 페이지 구조

```
/
├── /              (홈)
├── /portfolio     (포트폴리오 목록)
│   ├── /portfolio?category=web
│   ├── /portfolio?category=mobile
│   └── /portfolio?category=design
├── /portfolio/[id]  (프로젝트 상세)
└── /about        (선택 사항)
```

### 5.2 페이지별 기능

| 페이지 | 주요 요소 | 기능 |
|--------|---------|------|
| **홈 (/)** | Featured 프로젝트 카드 | 최대 6개의 주요 프로젝트 표시 |
| **포트폴리오 (/portfolio)** | 프로젝트 목록, 필터, 검색 | 전체 프로젝트 조회, 필터/검색 기능 |
| **상세 페이지 (/portfolio/[id])** | 프로젝트 상세 정보, 이미지, 링크 | 개별 프로젝트 전체 정보 표시 |

## 6. 설계 및 UX

### 6.1 시각적 디자인

- **색상**: 기존 스타터킷 CSS 변수 활용 (primary, secondary, muted 등)
- **레이아웃**: 반응형 그리드 (sm: 1열, md: 2열, lg: 3열)
- **컴포넌트**: shadcn/ui 기반 일관된 디자인
- **타이포그래피**: Geist 폰트 (Next.js 기본)

### 6.2 반응형 브레이크포인트

- **Mobile**: < 640px (1열 레이아웃)
- **Tablet**: 640px ~ 1024px (2열 레이아웃)
- **Desktop**: > 1024px (3열 레이아웃)

### 6.3 인터랙션

- 프로젝트 카드: 호버 시 그림자/스케일 효과
- 필터/검색: 즉시 반영 (쿼리 파라미터 이용)
- 페이지 전환: 자연스러운 과도 효과

## 7. 기술 구현 전략

### 7.1 Notion API 활용

```typescript
// 예시 구조
const projects = await notion.databases.query({
  database_id: process.env.NOTION_DATABASE_ID,
  filter: { property: "status", select: { equals: "완료" } },
  sorts: [{ timestamp: "last_edited_time", direction: "descending" }],
});
```

### 7.2 데이터 캐싱 전략

- **React Query**: ISR(Incremental Static Regeneration)과 함께 사용
- **캐시 시간**: 기본 60초 (설정 가능)
- **재검증**: 프로젝트 수정 후 수동 재생성

### 7.3 이미지 처리

- Notion 파일 URL에서 이미지 조회
- Next.js Image 컴포넌트로 최적화
- 대체 이미지 제공 (이미지 없을 경우)

### 7.4 SEO 최적화

- **메타데이터**: 동적 생성 (프로젝트별)
- **구조화된 데이터**: Schema.org JSON-LD
- **OG 태그**: 소셜 미디어 공유 최적화
- **Sitemap**: 자동 생성
