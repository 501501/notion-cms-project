# review-ui

현재 프로젝트의 UI/UX를 리뷰합니다.

## 사용법

```
/review-ui
```

## 검토 항목

1. **레이아웃 일관성**
   - 섹션 간 여백 균일성
   - 정렬 기준 (left, center, right)
   - 컴포넌트 재사용 패턴

2. **간격 및 정렬**
   - padding/margin 규칙 (Tailwind 스케일)
   - 텍스트 정렬과 요소 배치
   - 그리드 및 플렉스 구조

3. **Tailwind 사용 적절성**
   - 클래스 조합 최적화
   - 불필요한 커스텀 CSS 여부
   - 반응형 클래스 (sm, md, lg, xl)

4. **접근성(A11y)**
   - 버튼 시맨틱 HTML (`<button>` vs `<div>`)
   - 이미지 alt 속성
   - 폼 레이블 연결
   - 색상 대비율
   - 포커스 상태

5. **모바일 반응형**
   - 모바일/태블릿/데스크톱 레이아웃
   - 터치 타겟 크기 (최소 44x44px)
   - 뷰포트 메타 태그

## 프로세스

1. 프로젝트 구조 분석 (components, app 폴더)
2. 각 검토 항목에 대해 현재 상태 평가
3. **개선점만 간결하게 정리** (항목별 3-5줄)
4. 코드 수정 없이 제안만 제공
