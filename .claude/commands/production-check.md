# production-check

프로젝트를 배포하기 전 체크리스트를 수행합니다.

## 사용법

```
production-check
```

## 검토 항목

1. **console.log 제거 여부**
   - 개발 중 남겨진 콘솔 로그 확인
   - console.error, console.warn 제외

2. **환경변수 하드코딩 여부**
   - API 키, URL, 비밀번호 등 하드코딩 확인
   - .env.local, .env.example 설정 여부

3. **에러 처리 존재 여부**
   - try-catch 블록 미장착 API 호출
   - 콜백 함수 에러 핸들링
   - 네트워크 요청 에러 처리

4. **로딩 상태 처리 여부**
   - 비동기 작업 중 UI 피드백 (로딩 스피너, 버튼 비활성화)
   - skeleton, placeholder 활용

5. **SEO 메타 태그 여부**
   - title, description, og 태그
   - canonical URL
   - structured data (schema.org)

6. **불필요한 코드 존재 여부**
   - 미사용 임포트, 변수, 함수
   - 주석 처리된 코드
   - 테스트 코드, 임시 파일

## 프로세스

1. 프로젝트 전체 코드 스캔
2. 각 검토 항목별 문제점 확인
3. 문제점과 개선 제안 간단히 정리
4. 코드 수정 없이 보고서만 제공
